Структура arsc:

Package ...
  Type ...
  Config ...
    Entry ...
    Entry ...
    ...
  Type ...
  Config ...
    Entry ...
    Entry ...
    ...
  ...

Package звычайна адзін.

Кожны радок - гэта Entry ў Config.
Неабавязкова мець перакладзеныя ўсе entry. 

Strings: усе Entry з тыпам string.
   Астатнія тыпы - пустыя спасылкі, 

Arrays: усе Entry з тыпам string-array
   Астатнія тыпы - integer-arrays, items(list of references), colors
   
Plurals: усе entry

arrays зьмяшчае таксама часам цікавую інфармацыю, напрыклад, па якіх каардынатах пачынаць паказваць мапу. Але наўрацьці з мапай мае сэнс нешта мяняць, бо яно памятае апошняе месца.

Можна выдаляць усе блёкі Config з іншымі мовамі цалкам.

Тып RAW таксама можы утрымліваць нешта цікавае для перакладу, але яны зыхоўваюцца не ў .arsc, а як файлы ў .apk.
Але ў .arsc таксама на іх ёсьць спасылкі - ў тыпе RAW.
